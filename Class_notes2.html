A web page is a text document. The language used is primarily HTML.
How Coding Works
Coding is talking to the computer in a language the computer understands. It needs to be exactly right because computers are stupid.
Basic HTML Vocabulary
  • Tag: An HTML tag is what the computer takes direction from. Most tags have an opening tag. Some tags are called "void" tags and do not require a closing tag.
  • Element: An HTML element is everything within opening and closing tags.
  • Attribute: This is a property of an HTML element, put inside a tag.

Variables
Assigning variables clarifies code and prevents repetition.
Assignment statement:
Name = Expression. The equals sign means “assignment” rather than “equals”. Variables can vary throughout the program. You can reassign a variable at any time. You can use the variable on both sides of an assignment: days=days-1 is valid.
Strings
A sequence of characters. Strings are contained in single quotes or double quotes. They can be concatenated with +.
Indexing strings:
The first postion is position 0. the syntax is <string> [<expression>]
Selecting sub-sequences from strings: <string> [<expression>:<expression>].
Find:
Find is a method - a built in procedure in Python.
<string>.find(<string>) outputs first position in search string where target string appears. If it is not found it returns -1.
We can add a second parameter of a number, telling Python where in the string to look. Here is an example from the class:
example = "Wow! Python is great! Don't you think?"
first = example.find('!')
second = example.find('!', first + 1)
new_string = example[:first] + example[first+1:second] + example[second+1:]
print new_string # oops, I should probably replace the !s with periods
new_string = example[:first] +'.'+ example[first+1:second] +'.'+ example[second+1:]
print new_string
Triple quotes can be used for a multi-line string.

Classes

Classes are called like functions but they have a capiatlized first letter. When we call a class, we are running a function called __init__, which stands for initialize. When we initialize a class, we are creating a new instance of the class and creating space in memory for the instance.
A class is like a blueprint. It can be used to build many different things. Class Turtle specifies that the turtle can have size, color, direction, etc. But it doesn't specify what those qualities have to be.
Keyword "from"
We need "from" when importing a specific variable, class or function from a module. It gives directions for where to find the function/variable/class.
More about classes
A class allows us to create multiple instances and avoid repetition. Using a class creates abstraction. To define a class we use the keyword class and give it a title. Google Styleguide recommends the first letter of the title be capitalized. (Google Styleguide provides conventions on how to name things, and so much more.)

A class can have both data and methods.
Constructor
The __init__ function. Sets up the instance variables.
Instance Methods
Define functions that do things.
Class variables
Unlike instace variables, which may be different for each instance, class variables are the same for all instances of the class. All classes come with some pre-existing class variables. For example, __doc__, which gives documentation for the class.
Inheritance
A class can inherit qualities from another class, including methods. A child class can then overwrite methods. This is similar to CSS, in that in CSS, things are inherited unless they are overwritten. Inheritance and method overwriting are important because they prevent repetition (and create abstraction).
